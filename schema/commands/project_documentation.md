# Project Documentation (Quick Use)

**When:** Project completion, handoff, knowledge capture  
**Trigger:** "document this project" | "FOR[yourname].md"

## The Prompt

```
Write a detailed FOR[yourname].md file explaining the whole project in plain language:

- Technical architecture
- Codebase structure and connections
- Technologies used
- Why technical decisions were made
- Lessons learned:
  * Bugs encountered and fixes
  * Potential pitfalls and avoidance
  * New technologies used
  * Engineering thinking patterns
  * Best practices observed

Make it engaging - use analogies and anecdotes. Not boring textbook style.
```

## Copy-Paste Template

```
Create FOR[yourname].md for this project:

## Technical Architecture
[Explain in plain language]

## Codebase Structure
[How parts connect]

## Technologies
[What we used and why]

## Lessons Learned
- Bugs: ...
- Pitfalls: ...
- Best practices: ...

## Engineering Insights
[How good engineers think and work]

Use analogies and anecdotes. Make it memorable.
```

## The FOR[yourname].md Pattern

- **FOR YOU** - personalized, human-centered
- **NOT README.md** - those are for others
- **Living document** - captures journey, not just state
