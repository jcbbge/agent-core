# ADR-017: PROGRAM.md is JIT-compiled from executables, not documentation

**Date:** 2026-03-16
**Status:** Accepted

## Context

Agent operating contracts need to work across any project — including projects with no README, no walkthrough, no guides. The first iteration of `writing-program-md` assumed documentation files would be present and asked the human for context it couldn't find. This broke on every real test case.

The pattern Andrej Karpathy used in autoresearch: a single file that tells an agent how to work here without asking questions. The constraint: it must work even if the only things present are source files and git history.

## Decision

PROGRAM.md is generated by reading executables and inferring the operating loop from code — not by reading documentation. The skill must:
1. Read config files (deno.json, package.json, Cargo.toml, etc.) for tasks and entry points
2. Read executable files (CLI, main, worker) to derive what a complete unit of work looks like
3. Read schema/data files for what's frozen
4. Derive footguns from `fix:` / `revert:` / `hotfix:` commits in git history
5. Never ask the human for context it couldn't derive

The `## Operating Loop` section is required — not optional. If it can't be derived, the skill failed.

## Consequences

- PROGRAM.md now works on any project, including ones with no documentation
- Agents can bootstrap cold into a project and know what to do
- The contract drifts from reality only if the executables change without regeneration (acceptable — regeneration is cheap)
- Skill is more complex to run but produces higher-value output

## Alternatives Rejected

| Alternative | Why rejected |
|---|---|
| Ask human for context before reading code | Breaks autonomous model — human is meta-programmer, not co-pilot |
| Look for specific doc files (walkthrough.md, guides/) | Cannot guarantee their presence; fails silently on projects that don't have them |
| PROGRAM.yml (structured data) | Markdown is what agents actually read; structure adds friction without value |
| One-time generated file | Must be maintained by agents across sessions — Agent Log section enables append-only continuity |

## Resonance

The real insight: every project already contains its own operating manual — it's just written in code, not prose. The skill's job is translation, not invention.
