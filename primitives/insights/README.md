# Insights

One-line learnings distilled from tabs, articles, experiments.

## Format

```markdown
# [Topic] — [Insight]

**Source:** [original URL or context]
**Date:** YYYY-MM-DD
**Tags:** [relevant tags]

## Insight
[One paragraph max. What did you learn? What changes what you do?]

## Context
[Optional — why this matters, when to apply it]
```

## Examples

- "Tailwind purge removes unused styles in production — always check build output"
- "SolidJS createResource is async by default — don't wrap in createEffect"
- "SQLite WAL mode allows concurrent reads — enable for any multi-connection use"

## Naming

`[topic]-[insight-type].md`

Examples:
- `solidjs-async-pattern.md`
- `sqlite-wal-concurrent.md`
- `git-worktree-clone.md`
